疾病与baseline

最近在各种场合频繁看到 baseline，先是blade runner，然后是 mindhunter。

在软件开发中，回归测试( regression test)也是极其重要的一环。不过这个测试有效的前提是 baseline可靠。然后看条件变化后的 output是否符合期望。基本上分析都放在 validate变化及其导致的结果。

如果要思考 baseline是否有问题，就福柯了。 baseline是谁订的，疾病是否是一个被后天定义的东西，“疯子”是否是“正常人”政权底下的受害者。等等。

最近时刻感觉到自身的“偏离 baseline”，一个症状是感觉自己:I don’ t feel like myself，这不见得之前的你(baseline)至善至美，而是一种能力被剥夺的无力感，渐渐吞噬你存在的合理性。这种自我厌弃可能也是种自我改正的第一步。脑子在说：啊喂，有东西不对劲哦。

不对劲是真的，但是也不知道什么才是对劲，baseline是别人告诉我我应该是怎么样的这一个小刘，建立在以往经历和对我的推断上，从而形成了baseline小刘，而病后过往经历全部遗忘了，我对于自身的baseline建立在别人的嘴巴上，前一秒想跳出这个baseline宣布成为全新的小刘开始抬杠，后一秒有人告诉你，这都是你，逃不掉的。

遇到这种情况,有两种解决办法, update baseline是一种,与其 correct自己的行为（test suite），我不如更新baseline算了。这种“接受自己”也是心理治疗中的一个重要应用， feeling wrong？it’s Ok 就这样，接受自己存在的荒诞感，接受自己失灵，偏离变成契合，自我厌恶变成自我肯定。就这样。

想来人还是蛮精妙的一个设计的。
